# База знаний проекта (DecoRent)

Этот файл — “память проекта”: какие инструменты используем и как устроена логика.

## Стек (MVP)

- **Next.js (App Router)**: страницы лежат в `src/app/**/page.tsx`.
- **React + TypeScript**: типобезопасные компоненты.
- **Tailwind CSS**: быстрый UI, адаптив, единый стиль.
- **ESLint (Next preset)**: базовые правила качества кода.

## Текущая структура

- `src/app/layout.tsx`: корневой layout и metadata.
- `src/app/page.tsx`: главная страница.
- `src/app/catalog/page.tsx`: заглушка каталога (следующий этап — реальные данные/фильтры).
- `src/app/request/page.tsx`: заглушка формы заявки (следующий этап — отправка/валидация).
- `src/app/globals.css`: палитра и базовые глобальные стили.
- `src/components/*`: переиспользуемые UI‑компоненты.

## Палитра и дизайн‑принципы

- **Фон**: светлый “ivory” для воздушности (под свадебную тематику).
- **Акцент**: тёплая терракота (CTA, метки).
- **Второй акцент**: глубокий зелёный (kicker/иконки), чтобы балансировать “тепло”.
- Палитра задана CSS‑переменными в `globals.css` и используется через Tailwind‑цвета:
  - `bg-bg`, `bg-surface`, `text-text`, `text-muted`, `border-border`, `bg-brand`, `text-brand2` и т.п.

## Бизнес‑логика (как задумано по `spec.md`)

- Пользователь выбирает позиции в каталоге → формирует “список аренды” → оставляет заявку.
- **В MVP нет онлайн‑оплаты**: стоимость/залог/доставка подтверждаются вручную после заявки.
- Далее появится админ‑часть (NextAuth role=admin) и БД (PostgreSQL + Prisma).

## Git-подход

- Коммитим **важные устойчивые состояния**: каркас проекта, новая страница/секция, подключение инструментов.
- Маленькие атомарные коммиты проще откатывать и легче понимать историю.

## Следующие шаги (план на ближайшее)

- Реальный `/catalog`: категории, карточки, фильтры (пока можно на мок‑данных).
- Реальный `/request`: валидация формы и отправка заявки (email/в админку).
- Страница “Правила аренды” и “Спасибо за заявку”.
